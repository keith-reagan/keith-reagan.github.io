Setting up Lynis


Lynis is not a pre-installed package on CentOS or Ubuntu that helps out with system hardening by running a series of tests to check for vulnerabilites on the system. After running an audit the log produces some solutions to the issues it found and links to the error codes on the Cisofy website to assist with making improvements on your system. Below I've added the install process for both OSes with the comments proceeded by # and commands with $.

CentOS install:

					# login as su

# Check what other repositories are needed for install
$ dnf provides lynis

# Install epel-release repo if it isn't already
$ dnf install epel-release

# Update the cURL, NSS, openssl, and CA-certificates  packages
$ dnf update ca-certificates curl nss openssl

# Install Lynis
$ dnf install lynis

# Install latest community repo
$ cat >/etc/yum.repos.d/cisofy-lynis.repo <

Ubuntu install:

					# First check for updates and upgrade your packages if needed
$ sudo apt update
$ sudo apt upgrade - 32 packages upgraded

# Install HTTPS protocol
$ sudo apt install apt-transport-https

# Next, download the signing key from the central keyserver:
$ sudo wget -O - https://packages.cisofy.com/keys/cisofy-software-public.key | sudo apt-key add -

# To use Lynis only English version
$ echo 'Acquire::Languages "none";' | sudo tee /etc/apt/apt.conf.d/99disable-translations

# Install latest community repo
$ echo "deb https://packages.cisofy.com/community/lynis/deb/ stable main" | sudo tee /etc/apt/sources.list.d/cisofy-lynis.list
  
# Update your system again with the new repository
$ sudo apt update
$ sudo apt upgrade

# Finally, to install Lynis, use the following APT command:
$ sudo apt install Lynis

# Perform a show version to verify install
$ lynis show version

# Check if lynis is up to date
$ lynis update info

# Start auditing your system
# logs the audit information in the /var/log/lynis.log and audit report in /var/log/lynis-report.dat
$ sudo lynis audit system
