Basic Backup Shell Script


This is a short script I had created to backup files with the current year, month, day, hour and minutes timestamped onto the name

						$ grep Street datebook
#"grep" being the command, "Street" is the specific string I'm looking for in the file "datebook"

$ grep '^M' datebook
# Using RegEx inside the single qoutes the ^ anchor searches for the first letter in each line for an M, '^M' in databook

$ grep '000$' datebook
# Using RegEx to search for the string 000 at the end of each line with $ anchor
 
$ grep -m8 -v datebook
# I added in the -m8 so it would only search the first 8 lines and not flood the screen with results using the inverse match -v

$ grep '\/23\:' datebook
# The RegEx here show a string that starts with "/" using "\/" ("\" allowing to escape a special charact), then has "23" and ending with a ":" using "\:" in datebook

$ grep '\:[8][[:digit:]]\{2\}\-' datebook
# The "\:[8]" specifies the first part of the string needs ":8" which is the start of the area code after the persons name. "[[:digit:]]\{2\}" 
states that the next 2 characters need to be digits with the string ending with a "-" as the last character using "\-".

$ grep '[[:upper:]][[:lower:]]\{5\},[[:blank:]][[:upper:]]' datebook
# I think the ask was for a space after the comma so I went with this. "[[:upper:]]" for the first character in the string being an upper case letter,
#"[[:lower:]]\{5\}" for the next 5 characters to be lower case, ",[[:blank:]]" for a comma then space, and lastly [[:upper:]] for an upper case letter.

$ egrep '[[:digit:]]{2,3}[[:blank:]][[:upper:]]' datebook
# I had to go with egrep so I could use "{2,3}" after "[[:digit:]]" so it would look for two or three numbers in a row. Followed by "[[:blank:]][[:upper:]]" 
# for a blank space and uppercase letter

$ grep -n MA datebook
# The "-n" was used to display the line number of the entry that had the state of "MA" in the datebook file

$ egrep -m25 -v '(Street|ST)' datebook
# I used "-m" again to shorten the output, "-v" to show the inverse match of the Street OR ST - "(Street|ST)" results with having to use egrep for the OR.
