Basic Super Mario Bros ROM Hack


This info is assuming that you're editing SMB for the NES, no other game. However, there are plenty of tools out there to work on other games as well. These are just what I used.

First things first, to make sure that you have the right board and the right chip size you can go here

To edit the game sprites I used Tile Layer Pro. PLEASE note: only edit actual sprite tiles in this editor, if you try to edit what pretty much looks like colored "static" then you're messing with the actual game code. Chances are if you save over that your game won't work, ever.

Now that your game is all done you're ready to prep the files. You'll need to split that ROM file you were just editing into .chr and .prg files. For that you can use tniNES Utility. Now those files that you have created (we'll call the SMB.chr and SMB.prg) are 4kb and 32kb respectively. You'll need to use the copy command in the command prompt to enlarge the file. use the /b switch to add the file to itself. Example - copy /b smb.chr + smb.char + smb.chr + smb.chr smbbig.chr this will add the chr file to itself 4 times. You'll have to do it until it adds up to the size of the EPROM chip you're using. Doing the command above will make SMB.CHR 4kb into SMBBIG.chr 16kb.

Your EPROM chips will be between 120-200ns in speed. Make sure that you're getting a size that can contain the files. Each file has its own chip that it calls its home. I used the IC 27C256-15 EPROM 256K-Bit 150ns.There's different kinds, makers, etc. The model of this one is 27C256. This is the flavor of chip that AMD puts out. Note the last 3 numbers. 256 is your size. You'll see 128 (128kb), 512 (512kb), 1001 (1mb)etc. Just remember to make your .chr and .prg files those sizes. If you get 128kb chips then make sure that your files are 128kb. If not, you'll just get a gray screen when you go to play the cartridge.

Make sure to erase any chips you get using a UV EPROM eraser. I had a %100 failure rate with the 3 chips that I got that were used, thus the Jameco link.

The programmer that I used is no longer sold. Due to the age I could only get it to run on Windows XP, so take note of that. Select EPROM, then AMD, and then the model that you got. The model will be labeled on the chip as well so just look at it if you're not sure. Run the Blank test on your chip first, always make sure it's blank and your programmer can read it properly. Call your file up into the buffer, write, then verify. Label your chips with a piece of scotch tape or something of the like so you can remember what one is what.
